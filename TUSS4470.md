# TUSS4470 — Technical Deep Dive

*Texas Instruments TUSS4470 Transformer-Drive Ultrasonic Sensor IC with Logarithmic Amplifier. All figures from TI datasheet SLAS982 and application note SLDA058 unless noted.*

---

## 1. Overview

The TUSS4470 is an integrated analog front-end (AFE) IC for industrial ultrasonic sensing. It combines a direct-drive H-bridge transmit driver and a logarithmic receive amplifier in a single 4 × 4 mm WQFN package. It is not a complete system — it has no ADC, no MCU, and no digital output. It outputs an analog envelope signal that an external ADC must digitize.

**Target applications:** Industrial level sensing, flow metering (water, gas), proximity and distance measurement, object detection, pipeline monitoring, robotics obstacle avoidance.

**Frequency range:** 40 kHz – 1 MHz. This positions the TUSS4470 for industrial ultrasonic sensors — piezoelectric transducers in the sonar-through-air range and low-frequency liquid flow metering. It does not target medical imaging or NDT inspection (which require 1–30 MHz).

**Evaluation kit:** BOOSTXL-TUSS4470 BoosterPack paired with MSP-EXP430F5529LP LaunchPad. Software: TUSS44x0 EVM GUI (register configuration, echo display, power estimation). User's Guide: SLAU822A.

## 2. Integrated Functional Blocks

### 2.1 Transmit Driver (H-Bridge Pulser)

| Parameter | Value | Notes |
|-----------|-------|-------|
| Topology | H-bridge (full-bridge) | Bipolar drive across transducer |
| Max VPWR supply | 36V | Powers the H-bridge directly |
| Effective transducer swing | Up to 24V | Configurable via VDRIVE capacitor sizing |
| Drive modes | Direct-drive or pre-driver | Pre-driver mode for external power FETs |
| Frequency configurability | 40 kHz – 1 MHz | Set via SPI register |
| Pulse count | Configurable | Burst length set via SPI |
| Output impedance | Integrated; transformer not required | Transformer-less design is the key TUSS4470 differentiation |
| Current source | Internal capacitive charging | Capacitor on VDRV pin charges between bursts |

**Transformer-less design:** The predecessor TUSS4440 required an external transformer to achieve sufficient transducer voltage swing. The TUSS4470 adds an internal capacitive charge pump (VDRV capacitor) that accumulates energy between bursts and delivers it to the H-bridge at full VPWR swing. This eliminates the transformer from the BOM — a significant cost and board space reduction.

**Pre-driver mode:** For large transducers requiring more drive current than the internal H-bridge can supply, the TUSS4470 can be configured to drive external power MOSFETs. This extends the effective drive voltage and current range beyond the integrated stage.

**Comparison with WULPUS pulser:** The WULPUS MOSFET driver + +15V HV supply generates a unipolar +15V pulse. The TUSS4470's H-bridge is bipolar — it can swing the transducer from +VPWR to −VPWR, producing symmetric excitation with potentially higher acoustic efficiency and broader bandwidth. However, TUSS4470's frequency ceiling (1 MHz) is below WULPUS's transducer range (1–2 MHz).

---

### 2.2 Receive Path — Low-Noise Linear Preamplifier

The first receive stage is a fixed-gain, low-noise linear amplifier that conditions the echo signal before the bandpass filter and logarithmic amplifier. TI does not publish separate specs (noise figure, gain) for this stage; it is characterized as part of the full receive chain.

---

### 2.3 Bandpass / High-Pass Filter

| Parameter | Value | Notes |
|-----------|-------|-------|
| Filter topology | Switchable: bandpass or high-pass | Configurable via SPI |
| Bandpass center frequency | Configurable to transducer fc | Up to 500 kHz in bandpass mode |
| Above 500 kHz | High-pass filter mode | Bandpass bypassed; high-pass active |
| Configuration | SPI register | Set once at initialization |
| Purpose | Remove out-of-band noise | Improves SNR before log-amp |

**Bandpass mode (≤ 500 kHz):** The bandpass filter is tuned to match the transducer center frequency. This rejects low-frequency mechanical noise and high-frequency interference, maximizing SNR for narrow-band transducers.

**High-pass mode (> 500 kHz, up to 1 MHz):** The bandpass filter is bypassed; only a high-pass characteristic remains. This is a limitation for imaging applications where tight frequency selectivity improves clutter rejection.

---

### 2.4 Logarithmic Amplifier (Log-Amp / Time-Variable Gain)

The log-amp is the TUSS4470's most distinctive receive element.

| Parameter | Value | Notes |
|-----------|-------|-------|
| Gain range | 19 dB minimum to 113 dB maximum | **94 dB total dynamic range** |
| Gain type | Logarithmic compression | Output dB proportional to log(input amplitude) |
| Gain control | Continuous, via SPI register | Not stepped — smooth programmable compression |
| Output signal | Envelope of received echo | Demodulated; not raw RF |
| Output voltage | Analog (VOUT pin) | Scaled to 3.3V or 5V ADC reference |
| Output noise | 75–300 mV pk-pk | Depends on gain setting and supply |

**Log-amp vs linear TGC:** The TUSS4470 uses a logarithmic amplifier rather than a linear time-gain-compensated amplifier. A log-amp compresses the dynamic range of the received signal — high-amplitude echoes are attenuated relative to low-amplitude echoes — which is equivalent to applying a fixed compression curve to the entire echo window simultaneously. This is different from TGC, where gain increases monotonically with time (depth) to compensate for acoustic attenuation.

**94 dB dynamic range** is the specification that stands out. For comparison, the MSP430FR5043 USS_A PGA covers 37.3 dB (−6.5 to +30.8 dB). The TUSS4470 log-amp's wider dynamic range makes it more suitable for applications where echo amplitude varies over many orders of magnitude — e.g., detecting both near-surface reflections (strong) and deep back-wall echoes (weak) in a single acquisition without saturation.

**Output is demodulated envelope, not raw RF:** The VOUT signal is the amplitude envelope of the received echo, not the radio-frequency waveform itself. This is fundamental: you cannot reconstruct the original RF signal from the envelope, so post-processing techniques that operate on raw RF (matched filtering, coherent beamforming, Doppler extraction) are not possible with TUSS4470 alone.

---

### 2.5 Zero-Crossing Detection

The TUSS4470 includes a zero-crossing detector on the receive path. This is used for time-of-flight (ToF) measurement: the transition from transmit to the first echo zero-crossing gives the round-trip travel time. This output bypasses the log-amp and is available as a digital signal to the MCU, enabling precision ToF without ADC digitization of the full echo.

---

### 2.6 Configuration Interface

| Parameter | Value |
|-----------|-------|
| Interface | SPI only (no I2C) |
| Frame format | 1 RW bit + 6 address bits + 1 parity bit + 8 data bits (16 bits total) |
| Data direction | Full duplex (SDI/SDO) |
| Max SPI clock | Not specified in available sources; standard 1–10 MHz |
| Registers | Frequency, burst length, filter mode, gain, output scaling |

SPI is used for configuration only — register setup before acquisition. The actual echo data comes out on the analog VOUT pin, not SPI.

## 3. Power Consumption

| Mode | Current | Power @ 5V | Power @ 3.3V |
|------|---------|-----------|-------------|
| Active (burst + listen) | 11.5 mA | **57.5 mW** | 37.9 mW |
| Standby | 1.5 mA | 7.5 mW | 4.9 mW |
| Sleep | 350 µA | 1.75 mW | 1.15 mW |

**Active power context:** 57.5 mW active (at 5V) is higher than the entire WULPUS system (22 mW). It is comparable to the ADC10065 alone in pic0rick (68.4 mW). The TUSS4470 is not designed for wearable or battery-powered applications; its power budget is appropriate for mains-powered or industrial battery-backed devices.

**Standby power (7.5 mW)** is not negligible for battery operation. Between acquisitions, the TUSS4470 still draws 7.5 mW. For a 50 Hz duty cycle with 1 ms burst windows, the average power is approximately: 57.5 mW × 0.05 + 7.5 mW × 0.95 ≈ 10 mW — still above WULPUS's 22 mW full system.

**No ultra-low-power mode:** Unlike the MSP430FR5043 (450 nA standby), the TUSS4470 has no sub-mA standby mode. Duty cycling requires power gating the entire chip via its VDD pin.

## 4. Data Output and External Requirements

The TUSS4470 requires two external components to complete a system:

1. **External MCU:** For SPI configuration, burst triggering, and system control.
2. **External ADC:** For digitizing the VOUT envelope signal. Any ADC works — a 12-bit, 1 Msps ADC is typical (the envelope signal bandwidth is much lower than the echo RF bandwidth, because it is already demodulated).

**Data output type:** Analog envelope on VOUT pin. Not raw RF.

**ADC sampling rate required:** Because VOUT is an envelope (demodulated signal), its bandwidth is typically 2–5× lower than the transducer center frequency. For a 1 MHz transducer, the envelope bandwidth is ~200–500 kHz. A 1–2 Msps ADC is sufficient.

This is a key architectural difference: the TUSS4470 moves processing into the analog domain (log-compression + demodulation) and shifts the ADC requirement to a slower, cheaper converter. This reduces digital data rates and MCU processing load, but at the cost of losing access to the raw RF waveform.

## 5. Supply Voltages

| Supply | Range | Purpose |
|--------|-------|---------|
| VPWR | 5V – 36V | H-bridge drive + AFE supply |
| VDD_LOGIC | 3.3V or 5V | SPI interface, control logic |
| VDRV capacitor | Charged from VPWR | Energy reservoir for H-bridge burst |

The VPWR range (5–36V) makes the TUSS4470 flexible for different drive levels. For low-voltage operation (5V), the transducer swing is limited but adequate for many proximity sensing applications. At 12–24V (industrial supply), it can drive larger transducers with higher acoustic output.

## 7. Known Uses

**Primary documented uses:**
- Industrial ultrasonic water/gas flow meters (Badger Meter, Spire, similar)
- Level sensing (tank fill, pipe level)
- Proximity/obstacle detection in robotics
- Industrial range-finding

**Research uses:** Limited published academic use. The TUSS4470's analog-output-only architecture makes it less suitable for research requiring raw RF data or coherent processing. Research platforms (like pic0rick and WULPUS) that need raw RF waveforms require ADCs sampling at the transducer frequency, not at the envelope frequency.

**Wearable uses:** No published wearable medical ultrasound implementations using the TUSS4470 found in available literature. Its power consumption (57.5 mW active, 7.5 mW standby) makes it difficult to achieve multi-hour battery operation without significant duty cycling.

---

## 8. Constraints and Limitations

**1. Frequency ceiling at 1 MHz.** Industrial ultrasound transducers (40 kHz–1 MHz) are the design target. Medical imaging (5–15 MHz) and most NDT (2.25–10 MHz) are outside this range.

**2. No raw RF output.** The VOUT signal is a demodulated envelope. Raw RF reconstruction is not possible. Coherent processing (matched filtering, delay-and-sum beamforming, Doppler velocity measurement) cannot be performed on TUSS4470 data alone.

**3. No ADC.** Requires external ADC, adding BOM cost and board space.

**4. No MCU.** Requires external MCU for SPI control. The evaluation kit uses an MSP430F5529 LaunchPad.

**5. Active power too high for multi-day wearable use.** 57.5 mW active, 7.5 mW standby, vs WULPUS 22 mW total system.

**6. Single channel.** No multi-channel switching or array support integrated. Array imaging would require external mux chips.

**7. No time-gain compensation.** The log-amp applies a fixed compression curve. True TGC (monotonically increasing gain over the receive window) is not available from the integrated signal path.
