# MSP430FR5043 — Technical Deep Dive

*Analysis context: the MSP430FR5043 is the primary acquisition MCU in the WULPUS wearable ultrasound probe. All figures from TI datasheets and WULPUS published documentation unless otherwise noted.*

---

## 1. Overview

The MSP430FR5043 is a 16-bit ultra-low-power microcontroller from Texas Instruments in the MSP430FR5xx FRAM family. Its defining feature is the **USS_A (Ultrasonic Sensing Solution)** peripheral — a hardware-integrated subsystem combining a programmable pulse generator, a sigma-delta high-speed ADC, and a programmable gain amplifier, all sequenced by a dedicated state machine with DMA. This integration is what makes the WULPUS 22 mW system power budget achievable.

**Original design target:** Ultrasonic flow metering (water, gas, heat meters). The USS_A peripheral was engineered for transit-time flow measurement, where a transducer fires a pulse and the time-of-flight across a pipe determines flow rate. WULPUS repurposes it for wearable biomedical A-mode acquisition — a use case outside TI's intended scope, but within the hardware's physical capabilities.

---

## 2. USS_A Module — Programmable Pulse Generator (PPG)

The PPG generates the transmit excitation waveform under hardware control.

| Parameter | Value | Notes |
|-----------|-------|-------|
| Output frequency | 0–5 MHz | Configurable via register |
| Pulse count per burst | 0–30 | Register-limited; typical use 4–10 pulses |
| Duty cycle | Configurable | Pulse-on / pulse-off times independently set |
| Output driver impedance | 4 Ω | Integrated low-impedance PHY driver |
| Output voltage | 3.3V (MCU rail) | WULPUS HV PCB boosts to +15V via MOSFET driver |
| Timing granularity | µs-level | Register-based; not nanosecond-resolution |
| Timing programmability | Per-acquisition config | Set in 68-byte firmware config packet |

**Key constraint:** The PPG is a hardware state machine with a fixed instruction set. Unlike the RP2040 PIO (which is user-programmable at the assembly level), the PPG executes a fixed sequence (charge, pulse × N, quench). Custom pulse shapes (chirp, coded excitation, multi-frequency) are not achievable without additional external circuitry.

**Practical pulse count trade-off:** Fewer pulses = shorter transmit window = better near-field resolution. More pulses = higher acoustic energy = better SNR at depth. WULPUS papers typically use 4–10 pulses. At 30 pulses (max), the transmit window at 1 MHz is 30 µs, occupying ~23 mm of near-field dead zone in tissue (at 1540 m/s round-trip).

## 3. USS_A Module — SDHS ADC

The SDHS (Sigma-Delta High Speed) ADC is the digitizer for received echoes.

| Parameter | Value | Notes |
|-----------|-------|-------|
| Resolution | 12 bits | Unsigned output |
| Maximum sampling rate | **8 Msps** | Hardware ceiling; not firmware-configurable above this |
| Sub-rates supported | 8/10, 8/20, 8/40, 8/80, 8/160 Msps | Integer decimation from 8 Msps |
| ENOB | **Not published** | Critical missing spec; not in accessible TI documentation |
| Input range | 0 V – VREF | Single-ended; directly fed from PGA output |
| Anti-aliasing | Integrated in USS_A AFE | Exact bandwidth not published; optimized for flow metering |
| Data path | DMA → FRAM | No external bus required; fully on-chip |
| Latency | Sub-µs (inherent in sigma-delta) | No pipeline delay beyond oversampling ratio |

**The 8 Msps ceiling is hardware, not firmware.** The sigma-delta modulator's internal clock and digital filter topology set this limit. No register change or firmware modification can exceed it. This is the single most consequential constraint the MSP430FR5043 imposes on WULPUS.

**Axial resolution consequence:** At 8 Msps, time between samples = 125 ns. In soft tissue (c ≈ 1540 m/s), one sample = ~96 µm round-trip, or ~48 µm one-way depth increment. For comparison, pic0rick at 65 Msps achieves ~12 µm per sample. The MSP430FR5043 cannot support clinical B-mode probes (5–15 MHz) or standard NDT probes (2.25–10 MHz) — Nyquist requires fs > 2 × fc, so practical maximum fc ≈ 2–3 MHz.

**ENOB gap:** The absence of a published ENOB figure means actual dynamic range is unknown. Sigma-delta ADCs often achieve better ENOB than their nominal resolution at lower bandwidths (oversampling gain), but at 8 Msps the oversampling ratio is lower than typical sigma-delta applications. WULPUS papers do not report measured SNR or dynamic range.

## 4. USS_A Module — Programmable Gain Amplifier (PGA)

| Parameter | Value | Notes |
|-----------|-------|-------|
| Gain range (PGA only) | −6.5 to +30.8 dB | 46 discrete steps |
| Step size | ~0.81 dB per step | (37.3 dB range / 46 steps) |
| Adjustment time | µs-level | Register write; no slew time |
| External op-amp stage | +10 dB fixed | Power-gated between acquisitions |
| Total system gain | 40.8 dB (fixed during acquisition) | PGA held at max + external stage |
| TGC | **Not supported** | Gain cannot be varied mid-acquisition |
| Noise figure | Not published | Integrated AFE; no separate NF spec |
| Input impedance | Not published | Integrated; unknown transducer matching |
| Bandwidth | Not published | AFE bandwidth consistent with ≤8 Msps path |

**No TGC is a design-target consequence, not an oversight.** Flow meters measure a single reflector at a fixed, known distance (the pipe wall). Gain compensation across depth is irrelevant. For WULPUS's wearable monitoring use case (carotid at fixed depth, muscle at fixed depth), fixed gain is adequate. For imaging applications with variable depth or multiple targets, it is limiting. WULPUS-Pro addresses this with programmable TGC.

**46 discrete gain steps:** Each step is ~0.81 dB. For most biomedical monitoring use cases, 0.81 dB granularity is sufficient. For precise signal normalization or phantom measurements, the coarseness matters.

## 5. FRAM — Ferroelectric RAM

| Parameter | Value | Notes |
|-----------|-------|-------|
| Total size | 64 KB | On-chip non-volatile |
| Write endurance | ~10^15 cycles/byte | Effectively unlimited |
| Data retention | >100 years at room temp | Non-volatile without power |
| Access time | 125 ns typical | SRAM-speed access |
| Write speed | <1 µs per byte | No flash write penalty |
| Power (read) | Included in active-mode current | No separate read power |
| Power (write) | Included in active-mode current | No separate write power |

**Why FRAM matters for WULPUS:** FRAM allows the MSP430 to store acquired ADC samples at full speed without a separate external SRAM chip. 64 KB accommodates ~32,000 samples at 16 bits (about 80 full WULPUS frames at 400 samples each). The DMA fills FRAM during acquisition; the CPU then reads FRAM for SPI transfer to nRF52832. Because FRAM is non-volatile, the MCU can enter the deepest low-power mode (LPM4.5 — complete shutdown including RAM) between acquisitions and wake up with no state loss. This is not possible with SRAM-based designs.

## 6. DMA

| Parameter | Value | Notes |
|-----------|-------|-------|
| DMA channels | ≥6 (MSP430 standard) | Exact count varies by device variant |
| Burst mode | Yes | Multiple words per trigger, no CPU |
| Trigger sources | SDHS ADC, timers, USCI (SPI/UART) | Hardware-triggered without CPU |
| Transfer width | 8-bit, 16-bit, 32-bit | 12-bit ADC data packed as 16-bit |
| CPU interaction | Cycle-stealing (non-blocking) | CPU continues at reduced effective speed |
| Destination | FRAM, SRAM, SPI TX buffer | WULPUS: FRAM then SPI to nRF52832 |

**Automation chain:** USS_A state machine triggers → ADC completes sample → DMA interrupt → transfer sample to FRAM → loop until N samples → DMA sends FRAM block over SPI → CPU wakes from LPM to manage BLE. The CPU is not involved during the actual ADC conversion and DMA transfer, which is why standby current of 450 nA is achievable.

## 7. Clock System

| Component | Specification | Notes |
|-----------|--------------|-------|
| DCO (Digitally Controlled Oscillator) | 5.3–21 MHz tunable | No external crystal required |
| FLL (Frequency-Locked Loop) | Automatic frequency calibration | Locks DCO to reference without crystal |
| MCLK (Master Clock) | 16 MHz (WULPUS config) | CPU and main bus clock |
| SMCLK | 16 MHz or divided | ADC, SPI, timers |
| ACLK (Auxiliary Clock) | 32 kHz | RTC, LPM wake-up timers |
| Wake-up latency from LPM4 | <1 µs | Fast enough for precision timing |

**No external crystal needed:** The DCO + FLL combination achieves sufficient frequency accuracy for USS_A timing without a crystal oscillator. This reduces BOM, board space, and power. WULPUS firmware configures the DCO to 16 MHz and uses FLL for automatic calibration.

## 8. Power Modes

| Mode | CPU | MCLK | SMCLK | Typical current | Wake-up time |
|------|-----|------|-------|----------------|--------------|
| Active | Running | On | On | ~1.9 mA @ 16 MHz, 2.9V | — |
| LPM0 | Off | On | On | ~1.8 mA | <1 µs |
| LPM1 | Off | Off | On | ~1.5 mA | <1 µs |
| LPM2 | Off | Off | Off | ~2 µA | <1 µs |
| LPM3 | Off | Off | Off | ~500 nA | <1 µs |
| LPM4 + RTC | Off | Off | Off | **450 nA** | <1 µs |
| LPM4.5 | Off | Off | Off | ~35 nA | ~250 µs |

**WULPUS power strategy:** Between acquisitions at 50 Hz PRF, the MSP430 dwells in LPM4 + RTC (450 nA). The 20 ms inter-acquisition window (at 50 Hz) is dominated by LPM4 standby; the active window (~0.5–2 ms per acquisition) consumes ~5.5 mW. Time-averaged, the MSP430 contribution to the 22 mW WULPUS budget is small; the dominant power draw is the HV supply activation and op-amp stage during acquisition.

**LPM4.5 is possible but WULPUS does not use it:** LPM4.5 shuts down everything including I/O, requiring ~250 µs wake-up. This is too slow for 50 Hz PRF timing. At lower PRFs (e.g., 1 Hz continuous monitoring), LPM4.5 would reduce average current to ~35 nA, extending battery life dramatically.

## 9. Operating Voltage and Package

| Parameter | Value |
|-----------|-------|
| Supply voltage range | 1.8V – 3.6V |
| Typical WULPUS operating voltage | ~2.9V (single-cell LiPo, discharged) |
| Peak operating voltage | 3.7V (LiPo nominal) |
| Brownout threshold | ~1.6V |
| Packages | 64-pin LQFP, 48-pin LQFP, 32-pin VQFN |
| WULPUS package | 64-pin LQFP |

**Single-cell LiPo compatible without a regulator:** The 1.8–3.6V operating range covers the full LiPo discharge curve (3.7V nominal → 3.0V typical cutoff). WULPUS runs the MSP430 directly from the LiPo through a protection circuit, avoiding a linear or switching regulator. Eliminating the regulator saves the quiescent current that would otherwise be wasted (typically 50–200 µA for a low-dropout regulator).

## 10. Fundamental Constraints for Non-Flow-Metering Use

The MSP430FR5043 was designed for flow metering. Using it for wearable ultrasound imaging imposes these inherited constraints:

**1. 8 Msps ceiling is immovable.** Cannot be changed by firmware, register settings, or external clock. Limits usable transducer center frequency to ≤ 2–3 MHz.

**2. Fixed gain during acquisition.** The PGA gain must be set before the acquisition sequence starts. The USS_A state machine does not support mid-acquisition gain changes (which would be required for TGC — increasing gain over the receive window as echoes from deeper structures arrive).

**3. PPG limited to 30 pulses at up to 5 MHz.** Coded excitation (Golay pairs, chirp) is not achievable without an external arbitrary waveform generator driving the MOSFET.

**4. SDHS ADC is paired to the USS_A state machine.** The ADC cannot be used as a free-running ADC without the USS_A sequencer. It can be used standalone for non-US signals, but requires low-level register manipulation.

**5. Standalone SDHS standalone use is possible but undocumented.** The USS_A PPG can be disabled while the SDHS ADC is used as a general-purpose digitizer at 8 Msps. WULPUS firmware does not demonstrate this. Doing so would require TI register-level programming without a higher-level API.

## 11. USS_A vs a General-Purpose ADC + MCU Approach

| Dimension | MSP430FR5043 USS_A | External ADC + MCU (e.g., pic0rick) |
|-----------|-------------------|--------------------------------------|
| Integration | All-in-one | Multi-chip |
| Active power | ~5.5 mW (MCU only) | ADC10065: 68 mW alone |
| Sampling rate | 8 Msps max | 65 Msps (ADC10065) |
| ADC resolution | 12 bits | 10 bits (ADC10065) |
| Gain control | Fixed during acquisition | TGC, 12-bit DAC, runtime-adjustable |
| Pulse generator | Integrated (PPG) | Separate PMOD (MD1213 + TC6320) |
| Firmware complexity | Single-chip sequencer | Multi-peripheral coordination in PIO |
| Modification barrier | C register-level | PIO assembly (timing) + Python (host) |
| BOM count | 1 chip | 4+ chips (ADC, VGA, DAC, level shifters) |
| EDA tool | Any (open package) | KiCad (free) |

**Bottom line:** The MSP430FR5043 USS_A enables WULPUS's 22 mW total system power by integrating the entire analog signal chain on one die, eliminating the external ADC chip power budget. The price is fixed gain, 8 Msps ceiling, and the inherited constraints of a flow-metering design. For applications within the 1–2 MHz, fixed-depth, low-power envelope, it is the right chip. For anything requiring higher frequency, TGC, or arbitrary pulse shapes, an external ADC + general-purpose MCU is necessary.

## 12. What Would Be Needed to Go Beyond WULPUS's Current Implementation

**Higher sampling rate:** Requires a different chip. The MSP430FR5043 cannot be pushed past 8 Msps. An alternative: use the MSP430 for power management and timing control while adding an external high-speed ADC (e.g., AD9203 at 40 Msps, or ADS5400 at 65 Msps). This is essentially the architecture of WULPUS-Pro (external ADC, expanded channel count).

**TGC:** Requires either: (a) using the PGA's 46-step range in a stepped fashion between acquisitions (not true TGC, but stepped-gain depth gating), or (b) adding an external VGA (e.g., AD8331 or LMH6505) in front of the PGA. Option (b) adds power and board space. WULPUS-Pro implements a TGC amplifier as an external stage.

**More than 8 channels:** Requires cascading additional HV mux stages. The SPI bus can address multiple cascaded HV muxes. The MSP430 FRAM (64 KB) can buffer more channel data. The bottleneck shifts to BLE throughput (320 kbps) for wireless transmission.

**Arbitrary pulse waveforms (chirp, coded excitation):** Requires either the RP2040 PIO approach (cycle-accurate programmable GPIO) or an external FPGA/arbitrary waveform generator driving the MOSFET. The MSP430FR5043 PPG cannot generate non-rectangular or coded waveforms.

**Doppler processing:** Currently done in post-processing on the host. A dedicated Doppler DSP stage (e.g., quadrature demodulation followed by velocity estimation) could be implemented in FRAM-resident C code on the MSP430, but would consume CPU cycles and reduce available acquisition duty cycle.
